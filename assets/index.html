Prework Study Guide

HTML

  • HTML gives the content of a webpage structure
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • HTML attribute defines additional information about an element
  • Child elements are elements nested inside other elements (like this one)
  • Semantic HTML element is the name of the element describes its content and the role it plays on the webpage
  • Element refers to the entire element from opening tag to closing tag
  • Tag refers only to what appears inside the angle backets

CSS

  • A margin element indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • The box shadow CSS property adds shadow effects around an element's frame and is described by X and Y offsets relative to the element, blur, and spread radius, and color
  • .card - The period proceeding 'card' designates as a class selector and any HTML element assigned to the class, .card, will gain these property assignments from the CSS
  • There are (3) ways to style a webpage using CSS: Inline CSS, Internal CSS Stylesheet, or External CSS Stylesheet
  • Inline CSS: when developer wants to style an element directly in HTML file using a "style" attritbute -- useful for a quick and permanent change but less flexible than other two options because each inline style must be edited individually with any design change
  • Internal CSS: used to embed an entire stylesheet directly in HTML containing CSS rules for webpage in the head section of HTML file the rules apply to that page - can configure
  • External CSS: when styling is done completely seperate from HTML code, using < link > within the head of HTML code linking all styling to webpage -- the two code languages kept separate for developer ease

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: add command adds modifications in current working branch to the stage
  • -A flag indicates that we want to add all changes
  • staging area: informs Git that the changes will be added in the next commit
  • git pull origin main - *pull* command to receive a branch's modifications into the local environment *origin* indicates the source of the pull will be in the GitHub repo *main* indicates the branch
  • git pull: always run this command to ensure you have the latest project version on your local computer before starting your work!!

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Undefined type is inhabited by exactly one value, 'undefined'
  • Null is inhabited by exactly one value, 'null'
  • Boolean type represents a logical entity and inhabited by two values: true and false
  • console.log method is used for the output of a message to the web console by adding an argument
  • Console refers to a test environment that developers use to check out their code
  • Array: a single varible that is used to hold a group of data and are typically used to hold data that is related in some way
